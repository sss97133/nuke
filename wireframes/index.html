Novel Data Display Wireframes

Four approaches to surfacing Nuke's competitive advantage — data no other car listing site has. Each wireframe uses the same 12 sample vehicles with real-ish Nuke data: estimate ranges, price deltas, heat scores, deal labels, and data completeness.

Click any card to open that wireframe, or use the nav above.

A: The Bloomberg Card

Data-forward grid. Every card is a mini terminal readout with estimate range, delta, heat bar, data completeness, and deal label visible at idle.

max data density

B: The Signal Strip

Image-dominant card with a 3-cell signal footer. Price position + heat + data confidence as a color-coded strip below each card.

balanced

C: The Overlay

Novel data lives ON the image as transparent badges. Image stays huge. Deal label, heat, data completeness, and estimate are ambient overlays.

image-first

D: The Dashboard Row

Abandon masonry. Uniform rows with sortable columns: image, identity, bid, estimate range, delta, heat. Pure Bloomberg terminal.

power user

E: Badge Portals — Every Data Point is a Door

The novel interaction pattern. Every piece of data (year, make, body, transmission, location, deal score) is a clickable badge. Click it and a cluster panel expands inline showing related vehicles, estimate comparisons, and a "VIEW ALL" action. Click again or ESC to collapse. Hover shows depth count. This is what makes Nuke fundamentally different from any listing site.

core interaction
