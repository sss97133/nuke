# Dictionary: LLM Infrastructure Terms

## Models

**nuke** — Ollama model. DeepSeek R1 32B with full Nuke system prompt baked in. Knows schema, pipelines, domain, hard rules. 19.9 GB on SSD. ~15 tok/s on M4 Max. Use for interactive queries about the platform.

**nuke-agent** — Fine-tuned Qwen 7B trained on Nuke market intelligence data. Smaller, faster, specialized. Created via Modal training pipeline → GGUF export → Ollama import. Use for batch processing and user-facing interactions.

**Modelfile** — Ollama configuration file that defines a model variant. Specifies base model (`FROM`), system prompt (`SYSTEM`), and inference parameters (`PARAMETER`). Located at `yono/Modelfile.nuke` and `yono/Modelfile.nuke-agent`.

## Training

**QLoRA** — Quantized Low-Rank Adaptation. Fine-tuning technique that freezes the base model in 4-bit quantization and trains small adapter matrices (LoRA) on top. Enables fine-tuning 7B models on a single A100 GPU. Used in `modal_nuke_agent_train.py`.

**GGUF** — GPT-Generated Unified Format. Binary format for storing quantized LLM weights. Compatible with llama.cpp, Ollama, and LM Studio. The final output of the training pipeline. Quantization levels: Q4_K_M (recommended balance of size and quality), Q5_K_M (larger, slightly better), Q8_0 (largest, best quality).

**training example** — A JSON object with a `messages` array containing system/user/assistant turns. The model learns to produce the assistant response given the system and user messages. Stored as JSONL (one JSON object per line).

**cohort comparison** — Training example type. Shows top vs bottom performers within a vehicle segment and explains what drives the price difference. Generated by `export_nuke_market_intelligence.py`.

**gap analysis** — Training example type. Takes a specific underperforming vehicle, compares it against cohort leaders, identifies what's missing, and recommends specific fixes with estimated value impact.

**comment intelligence** — Training example type. Takes an auction comment thread and extracts sentiment, expert claims, red flags, and market signals. Teaches the model to read comments the way an expert would.

## Infrastructure

**Modal** — Cloud GPU platform at modal.com. Used for training (A100) and batch inference (T4). Pay-per-second billing. Key apps: `nuke-agent-train`, `nuke-vllm`, `yono-training`. Credentials in `nuke-sidecar-secrets` Modal secret.

**Modal volume** — Persistent storage on Modal. `yono-data` volume stores training data (`/data/nuke-agent/`, `/data/nuke-squarebody/`), checkpoints (`/data/nuke-agent-runs/`), merged models (`/data/nuke-agent-merged/`), and GGUF exports (`/data/nuke-agent-gguf/`).

**Telegram dispatch** — Notification system using `claude-notify` / direct Telegram API. Sends training progress, completion notifications, and error alerts. Bot token and chat ID stored in `.env` and Modal secrets.

**agent_tier** — Column on `vehicle_observations` table. Tracks which model produced the observation: `haiku`, `sonnet`, `opus`, `qwen-local`, `qwen-modal`, `nuke-agent`. Added in Phase 0.3.

**extraction_method** — Column on `vehicle_observations` table. How the data was produced: `direct-extract`, `re-extract-from-archive`, `enrichment`, `reconciliation`, `manual`. Added in Phase 0.3.

## Cost Terms

**A100 rate** — ~$3.15/hour on Modal. Used for training and GGUF export.

**T4 rate** — ~$0.59/hour on Modal. Used for batch inference (description discovery, reconciliation).

**local inference cost** — $0. M4 Max draws ~40W during inference. Electricity cost for 100K vehicles: ~$0.50.

**break-even point** — ~5,000 vehicles. Fine-tuning costs ~$15. After processing 5,000 vehicles locally (vs $0.003/each on Haiku), the investment is recovered.
