### Priority Scoring Algorithm

```typescript
interface ProviderScore {
  base_tier: number;           // 1-5 based on capabilities
  reputation_bonus: number;    // 0-20 points based on ratings
  completion_bonus: number;    // 0-15 points based on job history
  response_bonus: number;      // 0-10 points based on response time
  proximity_bonus: number;     // 0-10 points based on distance
  specialization_bonus: number; // 0-15 points if specialist
  total_score: number;         // Sum of all factors
}

function calculateProviderPriority(
  provider: Organization,
  job: ServiceRequest
): ProviderScore {
  
  // 1. Base tier (most important)
  const tierScore = provider.capability_tier * 20; // 20-100 points
  
  // 2. Reputation (quality matters)
  const reputationScore = Math.min(20, provider.avg_rating * 4);
  
  // 3. Completion rate (reliability)
  const completionScore = Math.min(15, provider.completion_rate * 15);
  
  // 4. Response time (speed)
  const avgResponseMinutes = provider.avg_response_time_minutes;
  const responseScore = Math.max(0, 10 - (avgResponseMinutes / 6));
  
  // 5. Proximity (convenience)
  const distanceMiles = calculateDistance(provider.location, job.location);
  const proximityScore = Math.max(0, 10 - distanceMiles / 5);
  
  // 6. Specialization (expertise)
  const isSpecialist = provider.specializations.includes(job.service_type);
  const specializationScore = isSpecialist ? 15 : 0;
  
  // Calculate total
  const totalScore = 
    tierScore +
    reputationScore +
    completionScore +
    responseScore +
    proximityScore +
    specializationScore;
  
  return {
    base_tier: provider.capability_tier,
    reputation_bonus: reputationScore,
    completion_bonus: completionScore,
    response_bonus: responseScore,
    proximity_bonus: proximityScore,
    specialization_bonus: specializationScore,
    total_score: totalScore
  };
}

// Premium Auto gets notified immediately (highest score)
```

## Database Schema

```sql
-- Organization capability tier tracking
CREATE TABLE organization_tiers (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  organization_id UUID REFERENCES businesses(id) NOT NULL,
  
  -- Current tier
  current_tier INTEGER NOT NULL CHECK (current_tier BETWEEN 1 AND 5),
  tier_name TEXT NOT NULL,
  
  -- Scoring
  capability_score INTEGER NOT NULL, -- 0-100
  staff_score INTEGER NOT NULL,      -- 0-100
  certification_score INTEGER NOT NULL, -- 0-100
  reputation_score INTEGER NOT NULL, -- 0-100
  total_score INTEGER NOT NULL,      -- Weighted average
  
  -- Performance metrics
  jobs_completed_count INTEGER DEFAULT 0,
  avg_rating DECIMAL(3,2),
  completion_rate DECIMAL(5,2),
  avg_response_time_minutes INTEGER,
  
  -- Financial
  annual_revenue_estimate DECIMAL(12,2),
  platform_fee_percentage DECIMAL(5,2),
  
  -- Next tier requirements
  next_tier INTEGER,
  next_tier_requirements JSONB,
  
  -- History
  tier_upgraded_at TIMESTAMPTZ,
  tier_downgraded_at TIMESTAMPTZ,
  
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Growth opportunities and recommendations
CREATE TABLE organization_growth_opportunities (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  organization_id UUID REFERENCES businesses(id) NOT NULL,
  
  -- Opportunity type
  opportunity_type TEXT NOT NULL CHECK (opportunity_type IN (
    'hire_staff',
    'buy_equipment',
    'get_certification',
    'expand_space',
    'add_service'
  )),
  
  -- Recommendation
  title TEXT NOT NULL,
  description TEXT,
  
  -- Investment
  investment_amount DECIMAL(10,2) NOT NULL,
  investment_type TEXT CHECK (investment_type IN (
    'one_time',
    'annual_recurring',
    'monthly_recurring'
  )),
  
  -- Return
  estimated_annual_revenue DECIMAL(10,2) NOT NULL,
  estimated_jobs_per_year INTEGER NOT NULL,
  roi_multiplier DECIMAL(5,2) NOT NULL,
  payback_period_months INTEGER,
  
  -- Priority
  priority_score INTEGER NOT NULL, -- 1-100
  recommended_phase INTEGER,       -- 1, 2, 3, etc.
  
  -- Status
  status TEXT DEFAULT 'recommended' CHECK (status IN (
    'recommended',
    'in_progress',
    'completed',
    'dismissed'
  )),
  
  -- Evidence
  missed_jobs_count INTEGER,
  missed_revenue_amount DECIMAL(10,2),
  evidence_job_ids UUID[],
  
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Job matching priority queue
CREATE TABLE job_matching_queue (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  service_opportunity_id UUID REFERENCES service_opportunities(id) NOT NULL,
  
  -- Organization
  organization_id UUID REFERENCES businesses(id) NOT NULL,
  priority_score INTEGER NOT NULL, -- Calculated score
  
  -- Timing
  notify_at TIMESTAMPTZ NOT NULL,  -- When to notify this provider
  notified_at TIMESTAMPTZ,
  expires_at TIMESTAMPTZ NOT NULL,  -- When their exclusive window ends
  
  -- Status
  status TEXT DEFAULT 'queued' CHECK (status IN (
    'queued',      -- Waiting to notify
    'notified',    -- Provider has been notified
    'viewed',      -- Provider viewed the opportunity
    'offered',     -- Provider sent an offer
    'expired',     -- Exclusive window expired
    'filled'       -- Job was filled by someone else
  )),
  
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_job_queue_notify ON job_matching_queue(notify_at)
  WHERE status = 'queued';
CREATE INDEX idx_job_queue_org ON job_matching_queue(organization_id);
```

## The Uber Analogy

### Traditional Model (Taxi)
- Fixed service area
- Same jobs for everyone
- No quality differentiation
- No growth incentives
- Manual dispatch

### N-Zero Model (Uber)
- **Tiered access** (UberX vs UberBlack)
- **Quality ratings** matter
- **Better providers** get better jobs
- **Clear path to upgrade** (better car = better earnings)
- **Intelligent matching** (algorithm optimizes)

**Key Difference**: It's not gatekeeping - it's **priority access**

Everyone CAN accept any job, but:
- Higher tier sees it first
- Higher tier gets notified faster
- Higher tier has exclusive bid window
- Lower tier still gets chance if no one accepts
